# 首页外竞赛展览台视觉重设计说明

## 目标

在不修改队员首页的前提下，重做首页之外的 JSP 页面，让整个系统拥有与首页同源但不复制首页的视觉语言。内页应更像大学生海报设计竞赛的作品目录、评审工作台和赛事运营台，而不是由 Bootstrap 卡片、紫色渐变和三块统计数字拼出的通用后台。

保护范围：

- `src/main/webapp/jsp/index.jsp` 完全不修改。
- `src/main/webapp/css/home.css` 完全不修改。
- `src/main/webapp/images/home/` 与首页脚本不修改。

## 设计判断

项目的真实使用场景是答辩现场与竞赛业务现场：老师在明亮的教室里浏览竞赛、作品和奖项；评委需要快速浏览作品并完成评分；管理员需要发布竞赛、管理公告和查看数据。界面因此采用明亮、清晰、带纸张与海报媒介感的轻展览风格，重点依靠内容层级和图像比例建立特色，而不是依靠大面积装饰。

品牌语气：年轻、创意、克制。

反模式：通用 SaaS 仪表盘、紫色渐变、全站相同白色圆角卡片、每页重复的大数字统计条、过度玻璃拟态、重复小型英文眉题。

## 视觉系统

### 字体与层级

- 中文正文使用系统无衬线字体，保证 JSP/Tomcat 环境不依赖网络字体。
- 页面标题使用更大的实心字重和较紧凑的行高；不再用渐变文字表达重点。
- 标题、正文、辅助信息至少形成三级对比：页面标题 > 内容标题 > 元数据。
- 重要数字只在确实有决策价值的场景出现，不再默认构成一排统计卡。
- 文字宽度控制在可读范围，按钮与表单标签不依赖全大写或过度字距。

### 色彩

- 深海蓝作为文字、导航和关键操作色，继承首页的稳定感。
- 青绿、海蓝、薄荷和黄色只作为作品标签、状态和局部强调色。
- 移除紫色作为主色，避免蓝紫渐变与“AI 管理后台”印象。
- 页面背景使用平静的浅蓝白底，不再叠加多层装饰性渐变。
- 卡片不使用“1px 浅边框 + 大范围柔和阴影”的模板组合；组件在边框或轻阴影之间做明确选择。

### 形状与媒介

- 普通信息块使用 10–16px 的适度圆角；按钮和状态标签可以使用胶囊形，但不把所有容器都做成胶囊。
- 作品缩略图、竞赛封面和证书画面承担主要视觉重量。
- 列表用目录行、作品墙和信息带区分，不使用一套尺寸完全相同的卡片铺满所有页面。
- 详情页使用“视觉主体 + 信息栏 + 流程/动作区”结构。
- 表单使用分区工作台结构，操作按钮集中在末端或侧栏，避免一个巨大的圆角白框包住全部内容。

## 页面家族

### 竞赛与公告目录

适用于 `competition_list.jsp`、`award_list.jsp`、`news_list.jsp`。

- 页面标题区显示标题、短说明和结果数量。
- 筛选与搜索合并为一条紧凑工具栏，避免独立的大白卡。
- 竞赛条目显示主题样张、竞赛名称、年度、状态、截止时间和进入详情动作。
- 公告和奖项使用横向目录行，突出日期、标题和内容摘要。

### 作品与评审队列

适用于 `submission_list.jsp`、`competition_works.jsp`、`score_list.jsp`、`judge_home.jsp`。

- 作品图像优先于装饰卡片，缩略图使用统一比例。
- 评委页先显示待评作品队列，再显示已完成评分记录；动作直接靠近作品。
- 点赞、分享、评分等动作使用轻量工具条，不用紫色按钮堆叠。

### 详情与展览页

适用于 `competition_detail.jsp`、`award_detail.jsp`、`certificate_view.jsp`、`team_detail.jsp`、`submission_detail.jsp`。

- 顶部使用清楚的返回路径和主标题。
- 主内容采用左右分栏或视觉主体优先的布局。
- 相关信息按“主题 / 时间 / 参与者 / 状态 / 下一步”组织，不以一张卡包住所有字段。
- 作品与证书预览保留足够留白，操作放在预览附近。

### 业务工作台

适用于 `competition_add.jsp`、`competition_edit.jsp`、`news_add.jsp`、`news_edit.jsp`、`score_input.jsp`、`team_create.jsp`、`profile.jsp`、`user_manage.jsp` 等表单和管理页。

- 表单按内容职责分为基本信息、规则/描述、提交动作等分区。
- 管理表格用清晰的表头、行间层级和行内动作，避免整表塞进视觉厚重的卡片。
- 管理员首页提供赛事运营入口；评委首页提供评审队列入口；两者不复用队员首页的参与者模块。

## 实现边界

- 继续使用 JSP、Bootstrap 和 Font Awesome，不引入新的前端框架。
- 共享规则集中在 `app-shell.css`；角色首页与作品墙等特殊布局放在 `role-home.css` 或明确的页面类中。
- 为页面添加语义化的 `.app-*` 结构类，逐步替代现有页面内联样式，但不进行后端接口重构。
- 真实作品图片继续使用现有 `image-data` 接口；没有竞赛封面时使用项目已有的海报样张作为视觉占位，不伪造业务字段。
- 保持现有 URL、表单字段、Servlet action 和权限判断不变。
- 所有动画遵循 `prefers-reduced-motion`，只使用轻微的进入和悬停反馈。

## 验收标准

1. `index.jsp` 与 `home.css` 的 Git diff 为空。
2. 首页外页面不再以紫色作为主视觉，不再出现重复的大面积渐变标题或三块统计卡模板。
3. 竞赛列表、作品列表、详情页、表单页、评委首页、管理员首页能看出不同页面家族，同时共享导航、字体、间距和状态色。
4. 作品/竞赛页面在有图片时优先展示图片，空数据时仍保留清晰的空状态和操作入口。
5. 全部现有 Maven 测试通过，WAR 可以重新打包；HTTP smoke test 继续覆盖队员首页、评委首页、管理员首页和核心列表页。
