# 全站前端统一与角色首页设计

**日期：** 2026-07-13  
**状态：** 已获确认，待编写实现计划

## 目标

以现有队员首页 `src/main/webapp/jsp/index.jsp` 与 `src/main/webapp/css/home.css` 为视觉基准，统一整个 JSP 前端的品牌体验，同时为评委和管理员增加独立的首页工作台。

队员首页是已完成的重点作品，内容、布局和视觉细节保持不变。评委与管理员首页复用同一套颜色、圆角、海报图片、轻玻璃层次和交互动效，但使用各自的任务内容与数据入口。

## 范围

### 包含

- 新增评委首页，面向“查看待评作品、进入评分、查看我的评分”。
- 新增管理员首页，面向“发布竞赛、管理竞赛、管理用户、设置获奖、发布新闻”。
- 更新首页入口分流：未登录用户和队员继续使用现有队员首页，评委和管理员进入各自工作台。
- 为 `src/main/webapp/jsp` 下的业务 JSP 页面增加统一的视觉壳层，包括登录、注册、个人中心、竞赛、队伍、作品、评分、获奖、证书、新闻和用户管理页面。
- 统一共享导航、页面背景、容器间距、标题层级、卡片、表单控件、按钮、表格、标签、提示和空状态。
- 保留现有 Servlet、表单 action、查询参数和业务流程，不做无关的后端重构。

### 不包含

- 不改队员首页的现有内容和 `home.css` 的既有表现。
- 不修改 React 进入页、上传文件、数据库表结构或业务权限规则。
- 不增加新的竞赛流程、评分规则或管理功能。

## 设计方向

### 视觉基准

沿用队员首页已经建立的视觉语言：

- 浅蓝色渐变背景和青绿色、粉色的柔和光晕。
- 深蓝色正文、蓝色强调色和低饱和的状态色。
- 适度圆角、半透明白色表面、海报图像作为主要视觉媒介。
- 衬线大标题 + 系统无衬线正文的层次组合。
- 150–250ms 的 hover/focus 反馈，支持 `prefers-reduced-motion`。

共享业务页面使用相同的色彩和形状，但密度更适合操作：标题区更紧凑，表格和表单保持清晰，避免把业务页面做成装饰性落地页。

### 队员首页保护策略

现有 `index.jsp` 继续使用 `body.home-page` 与 `home.css`。共享业务样式全部以 `body:not(.home-page)` 或角色首页专属 class 为作用域，避免全局覆盖队员首页。共享导航如需增加 class，只添加不影响原有视觉的标识；角色首页使用独立的角色样式覆盖导航外观。

## 页面与数据架构

### 首页路由

`IndexServlet` 继续作为 `/index` 的统一入口：

1. 加载现有竞赛列表和全局竞赛、队伍、作品、进行中竞赛统计。
2. 根据 Session 中的 `roles` 判断管理员、评委和参赛角色。
3. 管理员转发到 `/jsp/admin_home.jsp`。
4. 评委转发到 `/jsp/judge_home.jsp`。
5. 未登录用户和队员转发到现有 `/jsp/index.jsp`。

管理员首页额外使用现有用户服务获得用户总数。评委首页使用现有 `WorkDAO` 和 `ScoreService` 统计待评分作品（状态为已提交）与当前评委已评分记录，并提供最近待评分作品入口。所有统计都在 Servlet 中准备，JSP 只负责展示和安全转义。

### 评委首页

- Hero：`JUDGE STUDIO` 标签、评委问候、评分工作台主入口、我的评分次入口。
- 统计带：待评作品、已完成评分、进行中竞赛。
- 工作区：最近待评作品列表，显示作品标题、竞赛编号/入口和“开始评分”按钮；无数据时提供清晰空状态和评分工作台入口。
- 辅助入口：查看评分记录、往届获奖、新闻公告。

### 管理员首页

- Hero：`COMPETITION CONTROL` 标签、管理员问候、发布竞赛主入口、进入竞赛管理次入口。
- 运营统计：竞赛总数、进行中竞赛、参赛队伍、作品、用户。
- 操作区：竞赛管理、用户管理、获奖管理、新闻管理、证书管理。
- 内容区：展示最近/当前竞赛卡片，并在无竞赛时提供“发布第一个竞赛”入口。

### 全站业务页面

新增 `src/main/webapp/css/app-shell.css`，以已有 Bootstrap class 和项目现有页面 class 为兼容目标，覆盖但不改变业务 DOM 结构。需要新增的页面结构 class 仅用于角色首页和通用页面标题，不要求重写所有页面。

统一规则包括：

- 页面主体使用首页的浅色背景和窄内容宽度，导航与内容之间留出稳定呼吸空间。
- `.card`、`.card-custom`、`.form-card`、`.work-card`、`.cert-card`、`.team-grid-card` 等表面使用统一的浅色、圆角、细边框和轻阴影。
- `.form-control`、`.form-select`、`.btn`、`.badge`、`.alert`、`.table` 使用统一的 focus、hover、状态和间距规则。
- 表格在窄屏保持横向滚动，卡片网格在 720px 以下转为单列。
- 所有页面保留原有功能按钮、表单字段和链接，仅调整表现层。

## 错误与空状态

- 统计数据为空时显示 `0`，不因数据库返回空列表而 JSP 抛出异常。
- 首页竞赛列表为空时显示引导文案；管理员显示发布入口，评委显示评分工作台入口。
- 既有页面的错误/成功提示继续使用原 request attribute，统一为视觉一致的 alert。
- 角色没有对应数据时不隐藏主要入口，而是提供下一步行动链接。

## 可访问性与响应式

- 继续使用语义标题、按钮和链接；装饰性图片之外的图片保留有意义的 `alt`。
- 主要正文、按钮和表单文字维持可读对比度；focus 状态清晰可见。
- 页面在 1100px、720px 和 520px 附近适配：导航允许折叠/横向滚动，网格变为两列或单列，长表格可滚动。
- `prefers-reduced-motion: reduce` 下关闭角色首页的缩放/上浮动效，仅保留即时状态变化。

## 验证策略

### 自动验证

- Maven 编译和现有测试全部通过。
- 新增轻量 Servlet/JSP 结构检查，验证 `/index` 对管理员、评委、队员和未登录用户的目标页面分流，以及角色首页使用的关键 action 链接存在。
- 扫描所有 JSP，确认共享 `app-shell.css` 已加载（队员首页除外，避免改变既有首页）。

### 人工验证

- 分别用队员、评委、管理员账号访问 `/index`，确认页面内容不同但颜色、导航、卡片和按钮语言一致。
- 逐类打开登录/注册、列表、详情、表单、评分、获奖、证书、新闻和用户管理页面，确认没有被 CSS 隐藏的字段、错位的表格或失效按钮。
- 在桌面宽度和手机宽度检查导航、卡片、表单和表格；开启减少动效后检查内容仍然直接可见。
