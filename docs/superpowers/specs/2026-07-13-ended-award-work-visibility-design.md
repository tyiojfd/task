# 已结束比赛获奖作品查看设计

## 目标

比赛状态为“已结束”（`competition.status = 3`）后，任何已登录用户都可以查看该比赛中已有获奖记录的作品；未获奖作品维持现有查看权限。

## 范围

- 不允许未登录访问：仍由现有登录与权限过滤流程处理。
- 不修改数据库结构或已有获奖数据。
- 不改变评委、管理员、队伍成员查看作品的既有权限。
- 不开放同一比赛下的未获奖作品给非参赛者。

## 访问规则

`WorkAccessPolicy` 的作品查看权限判断按以下顺序执行，并同时供作品详情和图片读取使用：

1. 管理员或评委允许查看。
2. 作品所属队伍成员允许查看。
3. 比赛已结束且该作品存在 `award` 记录时，任意已登录用户允许查看。
4. 其他已结束比赛作品，仍仅允许该比赛参赛者查看。
5. 其他情况拒绝查看。

## 实现方式

在 `WorkServlet` 和 `ImageDataServlet` 中注入 `AwardService`，复用 `getAwardByWorkId(workId)` 查询获奖记录，再把结果传给 `WorkAccessPolicy`。作品详情和对应图片使用该放宽后的查看权限；点赞、取消点赞和分享继续使用不含“获奖例外”的原权限，避免把“可查看”扩大为“可互动”。不新增公开 URL，也不改变竞赛作品展厅的显示范围。

## 验证

- 未登录访问作品详情仍跳转登录。
- 非参赛已登录用户可以查看已结束比赛的获奖作品。
- 同一用户不能查看已结束比赛的未获奖作品。
- 非参赛用户不能因查看权限放宽而点赞、取消点赞或分享获奖作品。
- 原有队员、评委、管理员查看逻辑不回归。
