# 团队总结文档设计说明

## 目标

以用户提供的《团队总结.docx》为模板，生成一份可直接交付的团队总结文档，并将结果保存到 Windows 桌面。模板第一页为封面，必须保持原样，由用户自行填写封面中的团队信息。

## 约束

- 不修改模板第一页的文字、图片、排版、页边距和页码样式。
- 仅补充封面之后的正文内容。
- 不虚构团队编号、团队名称或成员姓名。
- 目标文件命名为“团队总结.docx”。
- “团队合影”部分保留明确的图片插入位置，便于用户后续替换。

## 方案

采用“以原文档为基底、保留封面、填充正文”的方案。直接复制模板文档，再在其已有的正文标题和空白位置填入内容，尽量复用模板的段落样式、字体、页边距和页码设置。与重新制作封面或将截图作为封面相比，该方案能够最大限度保留原始 Word 版式，并降低图片、字体和分页丢失的风险。

## 正文结构

1. 技术总结：介绍大学生海报设计竞赛系统的目标、Java Web 技术栈、MVC 分层架构、数据库设计、核心模块和安全措施，并总结系统开发成果。
2. 团队合作总结：说明垂直模块分工、协作流程、版本管理、阶段性成果、问题处理方式和团队收获。
3. 团队合影：保留居中的图片插入提示，不伪造照片。
4. 个人总结：按五类职责提供可编辑的个人总结段落，使用“队长、 副队长、杨祥博、队员B、队员C”等项目资料中已有的角色称谓；缺少真实姓名的部分不擅自补写姓名。

## 内容来源

正文内容依据项目根目录中的 `AGENTS.md`、现有开发进展记录和模板文档结构整理，覆盖截至 2026 年 7 月 14 日可确认的项目进度。内容采用正式课程设计报告语气，避免把未完成事项表述为已完成成果。

## 验证

- 生成后的文档能够被 `python-docx` 重新打开。
- 第一页仍包含模板封面和模板中的占位字段。
- 正文包含四个既定部分，且正文没有覆盖封面。
- 输出文件存在于桌面并可正常读取。
